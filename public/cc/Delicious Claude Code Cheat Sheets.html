Getting Started with Claude Code

1 / 5
Two modes, one window

Your terminal has different modes with limited visual clues for which one you're in. The shell is where you talk to your computer — run programs, move files, navigate folders. Claude Code is where you talk to Claude. You switch between them, but they look almost identical. The key difference is the prompt (that little symbol waiting for your input).

Installing Claude Code

Open your terminal and paste one of these commands:

Mac / Linux / WSL: curl -fsSL https://claude.ai/install.sh | bash
Windows PowerShell: irm https://claude.ai/install.ps1 | iex

That's it — nothing else to install.

Getting in and getting out

Type claude at your shell prompt and hit Enter. That's it — you're in. Claude will greet you and wait for instructions. To leave, type /exit or press Ctrl+C twice. You'll land back at your shell prompt. You can always get out. Your escape keys are always safe — but Claude's actions on your files are real, so review what it's doing before approving changes.

Where am I? What's here?

The terminal always operates inside a folder on your computer. By default, new terminal windows start in your home folder (~). Claude Code inherits that location when it starts. Use pwd to see where you are and ls to see what's in the current folder. Navigate with cd foldername to go deeper or cd .. to go up. Think of it like a file browser, but you type instead of click.

Start in your project folder

Claude needs a folder to work in. Always cd into your project directory before running claude. It reads the files around it to understand your project. If you start from the wrong folder, it's like handing someone a map of the wrong city.

# This is your shell. The $ is your prompt.
~$ cd my-project
~/my-project$ claude

# Now you're in Claude Code.
# The prompt changes — type normally.
claude> explain what this project does

# Claude responds, then waits for more.
claude> /exit

# Back to your shell.
~/my-project$ ▌
Panic button

If something looks wrong, something is running that you didn't expect, or you just feel lost:

Ctrl + C Cancel whatever is happening right now
Ctrl + C Ctrl + C Hit it twice to exit Claude Code entirely

These won't break anything. Use them liberally.

If you get lost

Look at your prompt. See $ or %? You're in the shell — type claude to start. See claude> or a Claude input area? You're already in Claude Code — just type your question.

How to get the most out of it
Be specific. Tell it what you want, where it is, and what done looks like. Vague prompts get vague results.
Work in small chunks. One task per prompt. Easier to review, easier to course-correct if it goes sideways.
Use CLAUDE.md. Run /init to create one. It's how you teach Claude about your project so you stop repeating yourself.
Review before approving. Read what it changed. Use git branches so you can always undo everything.

Best Practices for Claude Code

4 / 5
1

Plan first, then execute

Before any non-trivial task, ask Claude to outline its approach first. Read the plan, push back, then say "go." This one habit prevents most bad outcomes.

2

If you don't know, just ask

People forget this one. If you don't know how to do something, ask Claude to explain it. It's not just a code generator — it's a teacher, a debugger, and a sounding board.

3

Use /btw for side questions

Need a quick answer without derailing Claude's current task? Use /btw to ask a side question. It won't interrupt the flow of what it's working on.

4

Pipe in context, don't paste it

Instead of pasting logs or file contents, pipe them in: cat error.log | claude -p "explain". Cleaner, preserves formatting, and keeps your prompt focused.

5

Watch your context window

Check /context regularly. When it's getting full, /compact if you're still on the same task, or /clear if you're switching gears. Quality drops as context fills up.

6

Save context before clearing

Claude Code has no cross-session memory. Before you /clear, tell Claude to update its CLAUDE.md with anything important from the conversation. Otherwise it's gone.

7

Let Claude write its own memory

Have Claude write and maintain its own CLAUDE.md files. Periodically ask it to audit them for stale info and bloat. It re-reads this file constantly, so keeping it concise matters.

Claude Code Glossary

5 / 5
CLAUDE.md
A markdown file that acts as project memory. Claude reads it at the start of every session. Put your project conventions, preferences, and key context here so you don't repeat yourself.
Compacting
Compressing your conversation history to free up context space. Claude summarizes what's happened so far and discards the details. Triggered automatically or manually with /compact.
Context Rot
The gradual decline in quality during long sessions. As context fills with old back-and-forth, Claude's responses get less focused. The fix: /compact or /clear and start fresh.
Context Window
Claude's working memory for the current session. Everything you say, every file it reads, and every response it gives takes up space here. When it fills up, quality suffers.
Dangling Context
When Claude references files or decisions from earlier in a session that have been compacted or pushed out of the active window. Often shows up as sudden confusion about your project structure. Fix by re-stating key details or starting fresh.
Extended Thinking
When Claude pauses to reason through a complex problem before responding. Uses extra tokens but produces better results for hard tasks. Happens automatically.
Headless Mode
Running Claude Code non-interactively, e.g. in CI/CD pipelines or scripts. Use claude -p with piped input. No human in the loop — it runs and exits.
Hooks
Automated actions that run when specific events happen, like formatting code after a file is saved or running tests after changes. Set up with /hooks.
MCP (Model Context Protocol)
A way to connect Claude to external tools and services like databases, APIs, GitHub, or browsers. Managed with the /mcp command.
Permissions
Controls for what Claude can do without asking. Tools like file editing, bash commands, and web access can be set to allow, deny, or ask each time. Managed with /permissions.
Plan Mode
A way of working where you ask Claude to plan before executing. It thinks through the approach without making changes, so you can review and adjust before any code is written.
Skills
Like slash commands, but Claude can also trigger them automatically when relevant. Stored in .claude/skills/ with a SKILL.md file. The recommended way to extend Claude Code.
Slash Commands
Built-in commands starting with / that you type directly — like /clear, /compact, and /model. Custom slash commands (via .claude/commands/) have been replaced by skills.
Subagent
A separate Claude instance that works in its own context window and reports back. Used for parallel tasks or to keep specialized work from cluttering your main session.
Tokens
How text is measured. Roughly one token per word, though code and punctuation break differently. Context size and cost are both counted in tokens.
